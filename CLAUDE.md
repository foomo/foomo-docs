## Project Overview

Documentation site for the foomo stack (bestbytes), built with **Docusaurus 3.8** and deployed to https://www.foomo.org via GitHub Pages.

## Build & Development Commands

All commands run from the `foomo/` subdirectory. Uses **Yarn 4.9.4** (Corepack) and requires **Node >= 20**.

```bash
cd foomo
yarn install          # install dependencies
yarn start            # dev server with hot reload
yarn build            # production build to foomo/build/
yarn serve            # serve production build locally
yarn typecheck        # TypeScript type checking (tsc)
```

## Architecture

```
foomo-docs/
├── foomo/                     # Docusaurus site root (all yarn commands run here)
│   ├── docs/                  # Markdown/MDX documentation content
│   │   ├── general/           # Setup, tools, work practices
│   │   ├── frontend/          # CSS, React, debugging, performance
│   │   ├── backend/           # Go, HTTP, MongoDB, GoTsRPC
│   │   ├── devops/            # Kubernetes, monitoring, infrastructure
│   │   ├── projects/          # Foomo ecosystem projects (GoTsRPC, Keel, CMS)
│   │   └── security/          # Security guides across platforms
│   ├── blog/                  # Blog posts
│   ├── src/
│   │   ├── components/        # React components (GoPlayground, TypeScriptPlayground, Miro, Iframe)
│   │   ├── css/custom.css     # Theme customization (Infima CSS variables)
│   │   └── pages/             # Custom pages (homepage, awesome-software)
│   ├── docusaurus.config.js   # Site config (navbar, footer, Algolia search, Prism themes)
│   └── sidebars.js            # Auto-generated sidebars from directory structure
├── playgroundserver/          # Go-based playground backend
└── .github/workflows/         # CI: publish.yml (deploy on main push), validate.yml (PR build check)
```

## Key Conventions

- Each docs section has an `index.md` entry point referenced in `docusaurus.config.js` navbar items.
- Sidebars are auto-generated per section using `autogenerated` type in `sidebars.js`.
- Broken links cause build failures (`onBrokenLinks: 'throw'` in config) — always verify links work.
- The site supports Go syntax highlighting (`additionalLanguages: ['go']` in Prism config).
- Custom components (GoPlayground, TypeScriptPlayground) provide interactive code blocks; live codeblock theme is enabled for JS/JSX.

## CI/CD

- Push to `main` triggers build + deploy to GitHub Pages.
- PRs run a build validation check.
- Working directory in CI is `foomo/` — the Docusaurus site root, not the repo root.
